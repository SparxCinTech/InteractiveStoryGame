To implement an on-device Small Language Model (SLM) for adaptive NPCs in a psychological horror game, a core architecture integrating Perception, Cognition (SLM-driven), and Action loops is essential. This system should be supported by a dynamic memory system (with Short-Term and Long-Term components, including Semantic and Episodic structures) that records emotional and situational context, influencing an Artificial Psychosocial Framework (APF) for emotional modeling. The SLM acts as a reasoner for goal selection and high-level planning, which then interfaces with a planning algorithm like GOAP or HTN for detailed action execution. This adaptive approach, while more complex to implement than static rule-based systems, offers significant benefits in natural adaptation and emergent behavior, crucial for the nuanced emotional responses (betrayal, withdrawal, etc.) required in psychological horror.

Defining the Small Language Model (SLM) in Game AI

SLM as "Small Language Model"

In the context of game AI, particularly with advancements like NVIDIA's ACE (Avatar Cloud Engine) technology, "SLM" refers to a Small Language Model [14] [109]. These models are a more compact and efficient alternative to Large Language Models (LLMs), designed to run on-device, such as a player's PC, rather than relying on cloud-based processing.

This on-device capability is crucial for real-time interactions in games, minimizing latency and ensuring a seamless player experience. NVIDIA's ACE SLMs are specifically engineered to enable autonomous game characters that can perceive their environment, make decisions (cognition), and act in a manner that mimics human-like behavior.

Role of SLMs in Autonomous Game Characters

Small Language Models (SLMs) play a pivotal role in the development of autonomous game characters by providing the core intelligence for perception, cognition, and action [14] [134]. NVIDIA's ACE framework utilizes SLMs to allow NPCs to "perceive, plan, and act like human players."

Perception

Audio, visual, and game state processing

Cognition

High-frequency decision-making (8-13 sub-movements/sec)

Action

Strategic planning and reflection

Core Architecture: Integrating SLM, Memory, and Planning

Perception, Cognition (SLM), Action Loop

The Continuous Loop

The core architecture revolves around a continuous loop of Perception, Cognition (driven by an SLM), and Action, with Memory playing a crucial supporting role [14] [109].

1
Perception

Audio, visual, and game state information processing

2
Cognition

SLM analyzes input, considers goals and emotional state

3
Action

Execute decisions and generate speech via TTS

SLM for Decision-Making and Planning

SLMs are positioned as the central cognitive engine for decision-making and planning in next-generation autonomous game characters [14] [134].

Action Selection

Given the finite actions that can be taken in the game, the SLM chooses the best appropriate action from available possibilities [14].

Strategic Planning

Interface with larger LLMs for higher-level guidance or employ Chain-of-Thought (CoT) prompting for complex problem decomposition [109].

Reflection

NPCs consider outcomes of prior actions, allowing self-correction and improvement of decision quality over time [134].

Integrating LLM/SLM as a Reasoner for Goal Selection

Integrating a Small Language Model (SLM) as a "reasoner" for goal selection offers significant advancement over traditional AI methods [139] [200]. The architecture involves:

Reasoner Module (SLM)

  • • Processes memory information
  • • Considers personality traits
  • • Selects appropriate high-level goals
  • • Outputs natural language goals

Planner Module (GOAP/HTN)

  • • Receives selected goals
  • • Generates action sequences
  • • Uses automated planning techniques
  • • Executes concrete game actions

Memory System for Emotional and Situational Context

Simulated Emotional AI with Memories (Boyd Thesis Framework)

A robust memory system is foundational for NPCs to exhibit believable emotional responses. The PhD thesis by Reece Boyd, titled "Simulated Emotional AI with Memories for Games" [18], provides the Artificial Psychosocial Framework (APF) for implementing such systems.

Short-Term Memory (STM) and Long-Term Memory (LTM)

Short-Term Memory

Acts as a temporary buffer, holding information about the immediate environment and recent events. STM has limited capacity and duration [18].

  • • Player's current location
  • • Recent dialogue exchanges
  • • Objects interacted with
  • • Transient emotional reactions

Long-Term Memory

Stores information over extended periods, including significant experiences, learned knowledge, and deeply ingrained emotional associations.

  • • Episodic memories of major events
  • • Semantic knowledge about game world
  • • Emotional associations
  • • Character relationships

Semantic and Episodic Memory Structures

Episodic Memory

Stores specific, autobiographical events or "episodes" that the NPC has experienced. Each memory includes:

  • • What happened
  • • When it happened
  • • Where it happened
  • • Emotional impact
  • • Associated entities

Semantic Memory

Stores general knowledge and facts about the world, represented as a directed graph [18] where nodes correspond to concepts and edges represent relationships.

  • • Game mechanics knowledge
  • • Object properties
  • • Environment layout
  • • Social norms

Memory Dynamics

For a memory system to feel realistic, mechanisms like memory decay, consolidation, and emotional saliency are essential [18]:

Memory Decay: Gradual fading of memories over time if not accessed or reinforced
Consolidation: Transfer from STM to LTM based on emotional saliency
Emotional Saliency: Emotionally charged events are remembered better and influence behavior more strongly

Emotional Modeling and the Artificial Psychosocial Framework (APF)

Generating Emotions from Memories and Experiences

The Appraisal Process

The APF employs a sophisticated system for generating emotions in NPCs based on their memories and ongoing experiences [18]. Emotions are dynamically elicited through an appraisal process:

  1. NPC perceives event or retrieves memory
  2. APF evaluates against goals, needs, and personality
  3. Appraisal determines emotion type and intensity
  4. Emotional content influences current mood and decisions

Influence of Emotional State on NPC Behavior

Emotional Filters

The current emotional state acts as a filter through which the NPC perceives and interprets new information and recalls past memories [18].

Fear: Interpret ambiguity as threatening
Anger: Increase aggressive actions
Sadness: Lead to withdrawal or sulking

Behavioral Outcomes

Emotions influence goal prioritization and action selection, making behaviors emotionally congruent rather than purely logical.

Jealousy → Prioritize undermining rival
Despair → Suicide attempt behaviors
Betrayal → Complex revenge planning

Action Planning and Execution Algorithm

Goal-Oriented Action Planning (GOAP) and Hierarchical Task Networks (HTN)

GOAP (Goal-Oriented Action Planning)

Allows NPCs to generate plans at runtime based on current world state, available actions, and specific goals [38] [52].

Preconditions: Conditions for action execution
Effects: Changes to world state
Planning: A* search for action sequences

HTN (Hierarchical Task Network)

Works by decomposing high-level tasks into smaller sub-tasks until primitive, executable actions are reached.

Compound Tasks: High-level behaviors
Methods: Decomposition rules
Primitive Tasks: Directly executable actions

GOAP vs HTN Planning Example

GOAP Approach
Goal: KillPlayer
Plan: GetWeapon → MoveToPlayer → Attack
HTN Approach
Task: StalkPlayer
Method: [MaintainDistance, UseCover, AmbushIfOpportunity]

Algorithm: Integrating SLM, Memory, and GOAP/HTN

The iterative algorithm integrates SLM as a reasoning engine with memory systems and planning algorithms [139] [200]:

1

Perception and Memory Update (Interface)

NPC perceives game world, Interface updates "memories" data structure, converts raw data to natural language for SLM [200] [203].

Relevant memories retrieved from LTM/STM, emotional salience evaluated via APF [18].

2

Goal Selection (Reasoner - SLM)

SLM prompted with situational context: memory stream, emotional state, personality traits, role tags, potential goals [203].

Processes information, selects appropriate high-level goal (e.g., "Intimidate player" for Harasser with high aggression) [139].

3

Plan Generation (Planner - GOAP/HTN)

Selected goal + current world state passed to GOAP/HTN planner in "AP Problem" format [200].

Planner searches for action sequence using Domain Definition for available actions and logic [53] [203].

4

Plan Execution and Monitoring (Interface & NPC Controller)

NPC executes planned actions while Interface monitors execution. If world state changes unexpectedly, plan might become invalid [200].

If plan invalidated or completed, loop restarts. SLM can be consulted mid-plan for significant unforeseen events.

SLM's Role in Goal Generation and Action Selection

Goal Generation

SLM interprets context (perceptions, memories, emotional state) to formulate nuanced goals tailored to specific situations [109] [134].

Example: "Betrayed" emotional state + player vulnerability → "ConfrontPlayer" or "DemandExplanation" goals

Action Selection

SLM evaluates available actions based on utility AND alignment with personality, emotional state, and narrative impact [134].

Example: Choose between "LurePlayerToTrap" vs "CreateDiversion" for "IsolatePlayer" sub-goal

Integrating with the Drama System

Role Tags and Predefined Events

Harasser

Aggressive, threatening behaviors. Pursues and intimidates other characters.

Victim

Vulnerable, fearful responses. May trigger rescue or tragic events.

Negotiator

Mediates conflicts, can intervene in dramatic situations.

Role tags serve as high-level descriptors of an NPC's current narrative function. When combined with emotional states and significant actions, they trigger predefined dramatic events that maintain narrative cohesion while allowing emergent behavior.

Adaptive SLM vs. Static Rule-Based Systems

Complexity and Benefits Analysis

Implementation Complexity

Computational Cost: SLMs require substantial processing power for real-time inference on-device [152]
Design & Training: Requires robust training pipeline and high-quality datasets for psychological horror behaviors [156]
Integration Challenges: Complex interfaces with memory, planning, and drama systems [139]
Debugging & Control: Needs robust logging and XAI techniques to understand "thought process" [150] [156]

Key Benefits

Natural Adaptation: Processes rich inputs (emotional states, memories) for organic behavior changes [18] [139]
Emergent Behavior: Generates novel actions and strategies unanticipated by designers [150]
Character Differentiation: NPCs learn and adapt based on specific player interactions [142]
Enhanced Horror: Unpredictability and perceived intelligence amplify tension and fear [150]

Strengths and Limitations of Rule-Based Systems

Strengths

  • • Predictable & Controllable: Explicit behavior programming [148] [149]
  • • Simple Implementation: Straightforward for well-defined behaviors [57]
  • • Computationally Efficient: Less resource-intensive than SLM inference [150]
  • • Transparent Debugging: Clear conditional logic [57]

Limitations

  • • Inflexible: Limited to explicitly programmed behaviors [150]
  • • Poor Emotional Depth: Fails to capture complex human-like emotions [148]
  • • Repetitive Behavior: Predictable patterns reduce surprise [59]
  • • Combinatorial Explosion: Unwieldy for sophisticated AI [148]
Feature Adaptive SLM-based Systems Static Rule-Based Systems (RBS)
Adaptation & Learning High: Learns from experience, adapts to novel situations [149] Low: Follows predefined rules, limited adaptation [150]
Emergent Behavior High: Can generate unexpected, nuanced actions [150] Low: Behavior is predictable, follows scripts [150]
Emotional Depth High: Can model complex, evolving emotional states [18] Low: Typically models basic, static emotional responses [57]
NPC Unpredictability High: Enhances tension and replayability in horror Low: Patterns become predictable, reducing fear
Implementation Complexity High: Requires SLM integration, memory, training [152] Low to Medium: Easier to design and debug [57]
Computational Cost Higher: Requires on-device processing power [158] Lower: More efficient for simple behaviors [150]
Control & Predictability Lower: Harder to anticipate all behaviors, needs robust testing [156] Higher: Designer has explicit control over NPC actions [148]
Suitability for Horror High: Enables truly unsettling, reactive NPCs Medium: Good for scripted scares, less so for dynamic fear

Recommendation for Psychological Horror

For NPCs that must "adapt naturally to changing situations and act on their own emotions (e.g., betray, harass, murder, withdraw, sulk, suicide attempt)", adaptive SLMs offer a significantly more promising path despite higher complexity. The capacity for gradual emotional and behavioral shifts, driven by memory and contextual understanding, is far more aligned with the subtleties of psychological horror than static rule-based systems.

Implementation Considerations in Unreal Engine

On-Device SLM Performance and Optimization

Performance Challenges

Implementing an on-device SLM in Unreal Engine requires careful attention to performance and optimization. The primary challenge is ensuring SLM can run inferences quickly enough for real-time NPC decision-making without causing frame rate drops.

  • • Balance model capability with hardware resources (VRAM, CPU/GPU)
  • • Manage memory footprint of SLM and memory systems
  • • Maintain smooth player experience through continuous profiling

Optimization Techniques

Model Quantization:

Reduce precision of model weights to decrease size and inference time [158]

Pruning:

Remove less important connections or neurons to streamline model

Knowledge Distillation:

Train smaller model to mimic larger, more powerful one

Hardware Acceleration:

Utilize NVIDIA GPUs with Tensor Cores or dedicated AI hardware

Leveraging Unreal Engine's AI Tools and Plugins

Core UE AI Tools

Behavior Trees & Blackboards:

SLM acts as sophisticated "brain" setting high-level goals, Blackboard stores emotional state and SLM-generated goals

Environment Query System (EQS):

Evaluate positions based on cover, line of sight, distance to player for perceptual input

Gameplay Ability System (GAS):

Manage NPC actions, preconditions/effects, cooldowns, and resource costs

Planning Integration

GOAP Implementation:

Custom C++ implementation or community plugins like "Reasonable Planning AI Plugin" [53] [70]

Memory System:

UE data structures and serialization for STM/LTM, semantic and episodic memory components

External Integration:

Strong C++ support for integrating external SLM libraries and custom AI modules
